# Spec: AWS Infrastructure — Frontend

## Purpose

Defines S3 static website hosting per environment, runtime API URL injection via `config.js`, and React client integration.

## Requirements

### Requirement: S3 static website bucket per environment
The infrastructure SHALL provision one S3 bucket per environment (`staging`, `production`) configured for static website hosting. Each bucket SHALL have public read access for `GetObject`.

#### Scenario: Frontend is accessible via S3 website endpoint
- **WHEN** a browser navigates to the S3 static website URL
- **THEN** `index.html` SHALL be served as the root document

#### Scenario: Unknown paths fall back to index.html
- **WHEN** a browser navigates to a path like `/bicycles/1` directly
- **THEN** `index.html` SHALL be served (SPA routing support via error document set to `index.html`)

### Requirement: config.js injected at deploy time
Each environment bucket SHALL contain a `config.js` file at the root that sets `window.ENV_API_URL` to the ALB DNS name for that environment. This file is generated by CI and uploaded separately from the static build.

#### Scenario: Frontend reads API URL at runtime
- **WHEN** `index.html` loads in the browser
- **THEN** `config.js` SHALL be loaded before the React bundle, making `window.ENV_API_URL` available

#### Scenario: config.js is not cached by browser
- **WHEN** `config.js` is uploaded to S3
- **THEN** the S3 object metadata SHALL include `Cache-Control: no-store` to prevent stale API URLs

### Requirement: React app reads window.ENV_API_URL
The frontend API client (`frontend/src/api/client.js`) SHALL read the base URL from `window.ENV_API_URL` when defined, falling back to `""` (relative URL) for local development.

#### Scenario: API calls use ALB URL in production
- **WHEN** `window.ENV_API_URL` is set to the ALB DNS name
- **THEN** all API requests SHALL be prefixed with that URL

#### Scenario: API proxy works in local development
- **WHEN** `window.ENV_API_URL` is not set (local dev via Vite)
- **THEN** API requests SHALL use relative paths and be proxied by Vite to `localhost:8000`

### Requirement: index.html loads config.js before app bundle
`frontend/index.html` SHALL include `<script src="/config.js"></script>` in `<head>` before the Vite-injected bundle script.

#### Scenario: config.js is available before React renders
- **WHEN** the browser parses `index.html`
- **THEN** `config.js` SHALL execute before the React bundle initializes
